# Per-User Kite Credentials Implementation Guide

## Overview

Users now provide their own Zerodha API credentials for trading operations, while the application continues using YOUR credentials for market data (charts, signals, etc.).

### Architecture

```
┌─────────────────────────────────────────────────┐
│  Admin Credentials (from .env)                  │
│  Used for: Charts, signals, market data         │
│  └─ All users share this                        │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│  User's Kite Credentials (encrypted in DB)      │
│  Used for: Trading orders, account balance      │
│  └─ Each user's own API key                     │
└─────────────────────────────────────────────────┘
```

---

## Database Schema

### Table: `user_kite_credentials`

```sql
CREATE TABLE user_kite_credentials (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  user_id BIGINT NOT NULL UNIQUE REFERENCES auth.users(user_id),
  api_key TEXT NOT NULL,  -- encrypted
  access_token TEXT NOT NULL,  -- encrypted
  created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

**Supabase Setup:**

1. Go to Supabase Dashboard → SQL Editor
2. Run the above SQL to create the table
3. Enable RLS (Row Level Security):

   ```sql
   ALTER TABLE user_kite_credentials ENABLE ROW LEVEL SECURITY;

   CREATE POLICY "Users can see own credentials"
   ON user_kite_credentials
   FOR SELECT USING (user_id = auth.uid());

   CREATE POLICY "Users can insert own credentials"
   ON user_kite_credentials
   FOR INSERT WITH CHECK (user_id = auth.uid());

   CREATE POLICY "Users can update own credentials"
   ON user_kite_credentials
   FOR UPDATE USING (user_id = auth.uid());
   ```

## API Endpoints

### 1. Save User Credentials

**Endpoint:** `POST /user/kite-credentials/save`

**Request:**

```json
{
  "api_key": "your_zerodha_api_key",
  "access_token": "your_zerodha_access_token_24hr"
}
```

**Response (Success):**

```json
{
  "status": "ok",
  "detail": "Credentials saved and encrypted"
}
```

**Example cURL:**

```bash
curl -X POST http://localhost:8000/user/kite-credentials/save \
```


**Key Features:**

- ✅ Automatic encryption using `utils.auth.encrypt_credential()`
- ✅ Automatic decryption when retrieved
- ✅ Per-user data isolation via Supabase RLS
- ✅ Error handling with logging

---

### File: `backend/engine/auto_trader.py`

**Changes:**

1. `AutoTrader.__init__()` now accepts `user_id` parameter
2. Attempts to load per-user Kite credentials on initialization
3. Creates `self.user_kite` (per-user connection) if credentials found
4. Falls back to `self.kite` (admin) if no user credentials

**Helper Method:**

```python
def _get_kite_instance(self):
    """Returns per-user Kite or falls back to admin Kite"""
    if self.user_kite:
        return self.user_kite  # Use user's own credentials
    return self.kite           # Fall back to admin credentials
```

**Auto-trading flow:**

- Market data (signals, candles) → Uses admin Kite (`self.kite`)
- Trading orders (buy/sell) → Uses `_get_kite_instance()` (per-user if available)
- Account balance queries → Uses `_get_kite_instance()` (per-user if available)

```jsx
// component: CredentialsManager.jsx
import { useState } from 'react';
import { getAuthHeaders } from '../utils/auth';

function CredentialsManager() {
  const [apiKey, setApiKey] = useState('');
  const [accessToken, setAccessToken] = useState('');
  const [message, setMessage] = useState('');
  const [hasCredentials, setHasCredentials] = useState(false);

  // Check status on load
  useEffect(() => {
    checkCredentialsStatus();
  }, []);

  const checkCredentialsStatus = async () => {
    try {
      const res = await fetch('/user/kite-credentials/status', {
        headers: getAuthHeaders()
      });
      const data = await res.json();
      setHasCredentials(data.has_credentials);
    } catch (err) {
      console.error('Error checking credentials:', err);
    }
  };

  const handleSave = async (e) => {
    e.preventDefault();
    try {
      const res = await fetch('/user/kite-credentials/save', {
        method: 'POST',
        headers: getAuthHeaders(),
        body: JSON.stringify({
          api_key: apiKey,
          access_token: accessToken
        })
      });
      const data = await res.json();
      if (data.status === 'ok') {
        setMessage('✅ Credentials saved and encrypted!');
        setHasCredentials(true);
        setApiKey('');
        setAccessToken('');
      } else {
        setMessage(`❌ ${data.detail}`);
      }
    } catch (err) {
      setMessage(`❌ Error: ${err.message}`);
    }
  };

  const handleDelete = async () => {
    if (!window.confirm('Delete saved credentials? You must add new ones to trade.')) return;
    try {
      const res = await fetch('/user/kite-credentials', {
        method: 'DELETE',
        headers: getAuthHeaders()
      });
      const data = await res.json();
      setMessage('✅ Credentials deleted');
      setHasCredentials(false);
    } catch (err) {
      setMessage(`❌ Error: ${err.message}`);
    }
  };

  return (
    <div style={{ padding: '20px', border: '1px solid #ccc' }}>
      <h2>Kite API Credentials</h2>

      {hasCredentials && (
        <div style={{ ...padding: '10px', backgroundColor: '#e8f5e9', marginBottom: '10px' }}>
          ✅ Credentials saved. Your trades will use your own Kite account.
          <button onClick={handleDelete} style={{ marginLeft: '10px' }}>
            Change Credentials
          </button>
        </div>
      )}

      <form onSubmit={handleSave}>
        <div style={{ marginBottom: '10px' }}>
          <label>API Key</label>
          <input
            type="password"
            value={apiKey}
            onChange={(e) => setApiKey(e.target.value)}
            placeholder="Your Kite API key"
            style={{ width: '100%', padding: '8px' }}
          />
        </div>

        <div style={{ marginBottom: '10px' }}>
          <label>24-Hour Access Token</label>
          <input
            type="password"
            value={accessToken}
            onChange={(e) => setAccessToken(e.target.value)}
            placeholder="Your 24-hour access token"
            style={{ width: '100%', padding: '8px' }}
          />
        </div>

        <button type="submit" style={{ padding: '10px 20px', backgroundColor: '#1976d2', color: 'white' }}>
          Save Credentials
        </button>
      </form>

      {message && <p style={{ marginTop: '10px', fontWeight: 'bold' }}>{message}</p>}

      <div style={{ marginTop: '20px', padding: '10px', backgroundColor: '#f5f5f5' }}>
        <strong>How to get credentials:</strong>
        <ol>
          <li>Login to <a href="https://kite.trade" target="_blank">Kite Console</a></li>
          <li>Go to Settings → API Consents</li>
          <li>Copy your API Key</li>
          <li>Generate 24-hour access token</li>
          <li>Paste both above and save</li>
        </ol>
        <p><strong>Note:</strong> Your API key and token are encrypted before storing in our database.</p>
      </div>
    </div>
  );
}

export default CredentialsManager;
```

## Trading Flow

### Paper Trading

```
User starts auto-trader
    ↓
AutoTrader instance created with user_id
    ↓
Per-user Kite credentials loaded (if available)
    ↓
Signal detected → Buy order
    ↓
_execute_buy() called
    ↓
_get_kite_instance() returns per-user Kite (if has credentials)
    ↓
place_order() called with user's account
    ↓
Trade saved to database (tagged with user_id)
    ✅ Order placed on user's Kite account
```

### Real Trading

Same flow but:

- Uses user's credentials to place actual orders
- Account balance checked from user's account
- All P&L calculated on user's account

## Fallback Behavior

**If user has NOT saved credentials:**

```
Auto-trader still works (backward compatible)
    ↓
But trading orders will:
    ⚠️ Use admin credentials (shared account)
    ✅ Still track trades per user_id
    ❌ NOT recommended for production
```

**Warning logged:**

## Security

### Encryption

- All credentials encrypted using Fernet (symmetric encryption)
- Encryption key from `.env` `ENCRYPTION_KEY`
- Cannot decrypt without key

### Storage

- Encrypted credentials stored in Supabase
- Row-level security (RLS) prevents users from seeing others' credentials
- Database backups encrypted by Supabase

### Best Practices

✅ **DO:**

- Store only encrypted access tokens in database
- Log when credentials are accessed
- Alert user on credential changes
- Implement token refresh (24hr tokens)
- Use HTTPS for all credential transmission

❌ **DON'T:**

- Log credentials in plain text
- Store in client-side localStorage
- Transmit without HTTPS
- Keep old credentials after new ones added

## Troubleshooting

| Issue                            | Solution                                    |
| -------------------------------- | ------------------------------------------- |
| "No credentials saved" warning   | Ask user to save credentials in settings    |
| Orders not placing               | Check user's API key format in Kite console |
| "Invalid access token"           | Access token is 24-hour only, needs refresh |
| Database encryption error        | Regenerate ENCRYPTION_KEY in .env           |
| Permission denied on credentials | Check Supabase RLS policies                 |
